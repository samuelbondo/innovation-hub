## 1. Overview

The **Student Management System (SMS)** is a web-based platform for managing student registrations, admissions, departments, faculties, courses, and users. It supports four user roles with strict access control enforced on both the frontend and backend.

Key design principles:
- **No framework** — pure PHP, MySQL, and Vanilla JS
- **Rebrandable** — system name, tagline, footer all configurable from admin settings
- **Role-aware** — every page and API checks the user's role before serving data
- **Portable** — runs on any XAMPP/LAMP stack, no composer or npm required

## 6. Modules & Features

### 6.1 Student Registration (3-Step)
1. **Step 1 — Register:** Fill personal info, select faculty/department, set password
2. **Step 2 — Upload Docs:** Upload admission documents (Birth Certificate, ID, etc.)
3. **Step 3 — Done:** Registration number displayed, link to track admission

- Student ID auto-generated: `STU-YEAR-###` using a MySQL trigger
- Trigger uses `MAX(sequence)` to avoid duplicates after deletions
- Account created in `users` table with role `student`
- Status set to `Inactive`, admission_status set to `Pending`

### 6.2 Admission Workflow
```
Student Registers → Pending
Admin Reviews Docs → Under Review
Admin Decision → Approved ✅ or Rejected ❌
If Approved → student.status = Active → student can log in
```

### 6.3 Student Login Blocking
Students cannot log in if:
- `admission_status` ≠ `Approved` → "Your admission is still Pending..."
- `status` = `Suspended` → "Your account has been suspended..."
- `status` = `Inactive` → "Your account is inactive..."

### 6.4 Dashboard (Role-Aware)
- **Admin/Staff:** Stats (total/active/inactive students, departments, faculties), recent students table
- **Teacher:** My courses, department students
- **Student:** Student ID, department, year, status, enrolled courses, profile summary

### 6.5 Student Management (Admin)
- List with search + filter by department, year, status, admission status
- 🔔 Pending Signups banner — shows count, click to filter
- **NEW** badge on pending students
- Approve admission directly from list
- Student detail: edit info, manage admission, upload/delete documents, reset password, suspend/deactivate/delete

### 6.6 Course Management
- Add courses with code, title, department, year, semester, credits
- Assign teachers via dropdown per course
- **CSV Import** — bulk upload with validation and skip report
- CSV guide panel with column reference and downloadable template

### 6.7 User Management
- Add users with role: teacher, staff, admin
- Role filter tabs: All / Teachers / Staff / Admin
- Edit user details in modal
- Delete user with confirmation
- Reset password

### 6.8 Reports (Admin/Staff)
Four report types, all downloadable as PDF:
- **Student Report** — filterable by dept/year/status
- **Department Report** — student counts
- **Faculty Report** — dept and student stats
- **Admission Report** — all students with admission status

### 6.9 System Settings (Admin only)
- System name, tagline, footer text, contact info
- Saved to `settings` DB table
- Loaded dynamically on every page via `php/get_settings.php`
- Live preview on settings page

## 8. Authentication Flow

```
User submits login form
        ↓
php/login.php checks users table by email
        ↓ (if not found)
Check students table by student_id → get email → check users table
        ↓
password_verify() check
        ↓ (if student role)
Check admission_status and account status
        ↓ (if all pass)
Return user object → stored in localStorage
        ↓
shared.js reads localStorage on every page load
        ↓
Redirects unauthorized roles away from protected pages
```

## 9. Admission Workflow

```
1. Student registers via signup.php
   → students row: status=Inactive, admission_status=Pending
   → users row: role=student (cannot log in yet)

2. Admin sees 🔔 Pending Signups banner on view_student.php
   → NEW badge on pending rows

3. Admin opens student_detail.php → Admission tab
   → Can set: Pending / Under Review / Approved / Rejected
   → Can add a note (shown on public tracker)

4. On Approval:
   → admission_status = Approved
   → status = Active
   → Student can now log in

5. Student logs in → sees their dashboard with courses and profile
```

## 11. CSV Course Import

### Format
```csv
code,title,department,year_level,semester,credits
CS101,Introduction to Programming,Computer Science,1,1,3
IT201,Networking Fundamentals,Information Technology,2,1,3
```

### Rules
- Row 1 is the header — always skipped
- `department` must exactly match an existing department name
- `code` must be unique — duplicates are skipped with a reason
- `year_level`: 1–4, `semester`: 1–2, `credits`: 1–9
- Invalid rows are skipped and reported in the import result modal

## 12. PDF Reports

Reports use the browser's built-in **Print → Save as PDF** feature.

- Print CSS isolates only the report table — sidebar, topbar, buttons are hidden
- Report header shows title, generation timestamp, and record count
- All 4 report types supported: Students, Departments, Faculties, Admissions

## 13. Password Policy

### Signup
Passwords are scored on 5 criteria:
| Criterion | Points |
|-----------|--------|
| Length ≥ 8 | +1 |
| Length ≥ 12 | +1 |
| Contains uppercase | +1 |
| Contains number | +1 |
| Contains symbol | +1 |

| Score | Label | Color |
|-------|-------|-------|
| 1 | Very Weak | Red |
| 2 | Weak | Orange |
| 3 | Fair | Yellow |
| 4 | Strong | Teal |
| 5 | Very Strong | Green |

**Minimum required to register: Fair (score ≥ 3)**

### Admin Password Reset
Admin can reset any user or student password from their respective management pages. Minimum 6 characters enforced.

## 15. Known Limitations

- Auth relies on `localStorage` — suitable for LAN/intranet, not production internet deployment without HTTPS and server-side sessions
- No email sending — admission approval notifications are manual
- File uploads stored in `/uploads/` — no CDN or cloud storage
- No pagination on student/course lists — may be slow with 1000+ records
- Single-institution design — no multi-tenancy
